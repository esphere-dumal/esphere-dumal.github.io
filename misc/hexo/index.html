hexo|github|butterfly搭建美化博客

0x00 聊天局

在我刚进入UESTC的时候，做工作室招新题时涉及到了一些搭建博客的内容，也用hexo+github搭了一个粗鄙的博客，但是对于美化这一件事一直在当🕊。结果，口怕的事情发生了，孩子的第一台虚拟机炸了，里面数据也无了（我当时铁傻子，忘了尝试恢复），再加之我清理了一波自己github仓库的内容，顺手就把原来的博客库清了。至此，我第一个博客的所有内容全部虚无我也突然意识到我不需要博客，我需要一个能记东西的脑子，所以就打算彻底咕咕博客内容。

那么发生怎样更为口怕的事情让我决定重启博客计划呢？首先是隔壁🦈群转型成了一个科幻小说创造群人均新建文件夹，老🕊了，我觉得如果自己能写几篇短篇（那必然是做梦）可以顺手丢博客上；其次，是那个即将发售的游戏大概率是不会再跳票了，作为孩子成年后第一款3A游戏，那必然得写点东西丢博客；最后，好像是应该把平时学的东西以某种方式存储一波，不然就是学完就忘问题很大。

所以在GOGO狂牙大行动上线的那一个周末，我直接咕咕咕大行动开始嗯搭博客，在摸了大概10+h后，就有了你眼前现在这个博客。

这是一篇逼逼赖赖+教学的博客，上面逼逼赖赖完了，下面可以讲一些正经的东西了。

如果你用linux或者擅长使用windows,那么你应该可以跟着这篇博客轻松的搭建自己的博客。

当然博客内容是有时间局限性的，对于文中涉及内容，请以官方内容为准。

接着继续处理git

至此github相关的操作就完成了

0x02 安装hexo 搭建一个简陋的博客

hexo及相关组件安装

npm

npm | build amazing things (npmjs.com)

npm是一个随同Node.js安装的包管理工具，而hexo是基于Node.js实现的，所以自然需要npm进行安装管理。

hexo简单使用

首先hexo init，此时会自动去githubclone一个起步版本。

yaml

.yml是一种类似json的配置文件

与json一样大小写敏感

与json使用{}嵌套表示层级关系不同，yaml缩进表示层级关系

json中无注释，yaml中可用#标识注释行

其中_config.yml是配置文件，对其修改，添加deploy条目相关信息

此时，已经可以通过hexo g && hexo d进行部署

目录结构

博客根目录下source文件夹中_post中存放文章 .md文件

同时source下也会存储tags catelogeries等页面信息

文章标识

在每一篇.md文章的开头部分以---分割一个区域，里面记录一篇文章的相关信息

发布一篇文章

将文章前加上相关标识内容，接着就可以丢进/source/_post/ 就行了，最后hexo g && hexo d

0x03 安装butterfly主题美化博客

接着你会在博客根目录下theme文件夹中多出一个butterfly,将其中的_config.yml文件拷贝至根目录下改名为_config.butterfly.yml然后就可以对里面的内容进行修改实现自定义。
