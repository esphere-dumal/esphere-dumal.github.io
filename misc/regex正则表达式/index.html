正则表达式

简介

正则表达式是一种文本模式，描述了字符串匹配模式，包括普通字符和特殊字符（元字符）。

一个例子

第二个例子

^[0-9]+abc$

^为匹配输入字符串开始的位置

[0-9]+匹配多个数字，[0-9]匹配单个数字，+匹配多个。

abc$匹配字母abc并以abc结尾，$为匹配输入字符串的结束位置。

比如说一个数据库的用户名可以用一下正则:^[a-z0-9_-]{3-15}$

[] 中为匹配内容 {}中为字符长度 ^为起始标志$为终止标志

作用

测试字符串内的模式

替换文本

基于模式匹配提取子字符串

比如提取HTML 中有用信息

语法

构造正则表达式的方式和数字表达式类似，用多种元字符和运算符将小的表达式结合在一起组成更大的表达式。

组件

单个字符、字符集合、字符范围、字符间的选择以及这些组件的任意组合。

普通字符：a-z \ 1-9 \ - _ 等等字符

特殊字符: 元字符

普通字符

没有显示指定为元字符的字符，分为可打印字符、不可打印字符，包括了所有大小写字母，所有数字、所有标点符号、一些不是元字符的符号。

可打印字符

[,,,,,,]

匹配括号中所有字符

比如[aeiou]将匹配所有aeiou字母

[^……]

匹配所有非括号中的字符

[A-Z]

匹配一个区间

[\s]

匹配所有空白符，包括换行

[\S]

匹配所有非空白符，包括换行

[\s\S]

匹配所有字符

[\w]

匹配字母、数字、下划线。等价于[A-Za-z0-9_]

不可打印字符

字符 描述
\cx 匹配由x指明的控制字符。例如， \cM 匹配一个 Control-M 或回车符。x 的值必须为 A-Z 或 a-z 之一。否则，将 c 视为一个原义的 ‘c’ 字符。
\f 匹配一个换页符。等价于 \x0c 和 \cL。
\n 匹配一个换行符。等价于 \x0a 和 \cJ。
\r 匹配一个回车符。等价于 \x0d 和 \cM。
\s 匹配任何空白字符，包括空格、制表符、换页符等等。等价于 [ \f\n\r\t\v]。注意 Unicode 正则表达式会匹配全角空格符。
\S 匹配任何非空白字符。等价于 [\f\n\r\t\v]
\t 匹配一个制表符。等价于 \x09 和 \cI。
\v 匹配一个垂直制表符。等价于 \x0b 和 \cK。

不可打印字符主要涉及页面控制

特殊字符（元字符）

这类字符有特殊含义，如果要直接表达这些字符本身的意思，需要加上转义\

特殊字符 描述
$ 匹配字符串结尾位置。如果设置了 RegExp 对象的 Multiline 属性，则 $ 也匹配 ‘\n’ 或 ‘\r’。
( ) 标志一个子表达式的开始和结束位置
* 匹配前面子表达式零次或多次（简单来说就是所有）
+ 匹配前面子表达式的一次或多次
. 匹配处换行符\n的任意单字符
[ ] 标志一个中括号表达式的开始结束
？ 匹配前面字符串的零次或一次（最多一次），或指明一个非贪婪限定符
\ 转义
^ 匹配字符串开始的位置。如果在方括号中使用，则表示非，不接受方括号中字符匹配。
{ } 标识限定表达式的开始结束
| 或，两项之间二选一

限定符

限定符用来指定正则表达式的一个给定组件必须要出现多少次才能满足匹配

字符 描述
* 匹配前面的子表达式零次或多次。
+ 匹配前面的子表达式一次或者多次
？ 匹配前面子表达式零次或一次
{n} 匹配确定的n次
{n,} 至少匹配n次
{n,m} 匹配n~m次

[1-9][0-9]*匹配一个正整数。第一个数字不为零，后面有若干个数字。

注意，*和+限定符都是贪婪的，他们会尽可能的多匹文字。在后面加上?实现非贪婪或最小匹配。

定位符

定位符可以将正则表达式固定到行尾或者行首。

^ $将表示字符串开始和结束

\b 表示单词的前或后边界，即字与空格间的位置。

\B表示非单词边界

注意，定位符和限定符不能同时使用。

选择符

( )

用圆括号 () 将所有选择项括起来，相邻的选择项之间用 | 分隔。

() 表示捕获分组，() 会把每个分组里的匹配的值保存起来， 多个匹配值可以通过数字 n 来查看(n 是一个数字，表示第 n 个捕获组的内容)。

/([1-9]([a-z]+))/g

?:

但用圆括号会有一个副作用，使相关的匹配会被缓存，此时可用 ?: 放在第一个选项前来消除这种副作用。

其中 ?: 是非捕获元之一，还有两个非捕获元是 ?= 和 **?!**，这两个还有更多的含义，前者为正向预查，在任何开始匹配圆括号内的正则表达式模式的位置来匹配搜索字符串，后者为负向预查，在任何开始不匹配该正则表达式模式的位置来匹配搜索字符串。

?=

exp1(?=exp2) 查找exp2前面的exp1

?<=

exp1(?<=exp2)查找exp2后面的exp1

?!

exp1(?!=exp2)查找后面不是exp2的exp1

?<!

exp1(?!<=exp2)查找前不是exp2的exp1

这一部分可以进一步了解先行断言和后行断言 lookahead lookbehind/

反向引用

对一个正则表达式模式或部分模式两边添加圆括号将导致相关匹配存储到一个临时缓冲区中，所捕获的每个子匹配都按照在正则表达式模式中从左到右出现的顺序存储。缓冲区编号从 1 开始，最多可存储 99 个捕获的子表达式。每个缓冲区都可以使用 \n 访问，其中 n 为一个标识特定缓冲区的一位或两位十进制数。

可以使用非捕获元字符 ?:**、?=** 或 ?! 来重写捕获，忽略对相关匹配的保存。

确定重复出现的句子

/\b([a-z]+) \1\b/ig

正则表达式后面的全局标记 g 指定将该表达式应用到输入字符串中能够查找到的尽可能多的匹配。

表达式的结尾处的不区分大小写 i 标记指定不区分大小写。
