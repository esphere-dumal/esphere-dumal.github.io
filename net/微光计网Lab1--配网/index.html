微光计网Lab1–配网

我们这里简单复习一下分层结构

OSI参考模型：应用层 表示层 会话层 传输层 网络层 数据链路层 物理层

TCP/IP参考模型：应用层 传输层 网际层 网络接口层

理论知识

  • gateway：在网络层实现网络互联。可用于广域网和局域网互联。用于不同主机、系统、协议之间的转化（大概
  • IP地址：互联网上每一个网络和主机都有一个逻辑地址，以此屏蔽物理地址的差异。
  • netmask：不能单独存在，必须结合IP地址一起使用，将某个IP地址划分为网络地址和主机地址两部分。
  • DNS：domain name system，互联网一项服务，将域名和IP地址相互映射的一个分布式数据库
  • DHCP：动态主机配置协议，是一项局域网的网络协议。在大型局域网中集中管理分配IP地址，是网络中的主机动态的获取IP地址、Gateway地址、DNS服务器地址。提升地址的使用率。

网络适配器+VMware

  • 网路适配器（网卡）：拥有一个独一无二的MAC地址，属于OSI模型中的数据链路层。使用户可以通过电缆或无线进行互联。

  • MAC地址：media access control address，一个用来确定网络设备位置的位址。

Vmware提供了三种网络工作模式

  • 桥接：VMnet0。将主机网卡和虚拟机虚拟的网卡利用虚拟网桥进行通信。物理主机被虚拟成一个交换机，所有桥接模式的虚拟机都链接到这个交换机的一个接口上，物理主机也插在这个交换机上。桥接模式下，虚拟机的IP地址需要和主机在用一个网段，如果需要联网，那么网关与DNS需要主机网卡一致。
  • NAT：VMnet8。通过虚拟NAT设备和DHCP服务器使虚拟机可以联网。主机网卡与虚拟NAT设备直接连接，然后虚拟设备与虚拟DHCP服务器一起连接在虚交换机Vmnet8上，实现虚拟机联网。而虚拟网卡VMware Network Adapter VMnet8用于实现主机与虚拟机的通信。
  • Host-Only：VMnet1。其实就是NAT模式去掉了虚拟NAT设备，使用VMware Network Adapter VMnet1虚拟网卡连接VMnet1虚拟交换机来与虚拟机通信。Host-Only模式将虚拟机与外网隔开，使得虚拟机成为一个独立的系统，只与主机相互通讯。如果要使得虚拟机能联网，我们可以将主机网卡共享给VMware Network Adapter VMnet1网卡，从而达到虚拟机联网的目的。

在本机上可以找到这两玩意，但是找不到VMnet0
