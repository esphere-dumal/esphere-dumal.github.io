attacklab

这一部分涉及一些基于缓冲区攻击pwn的知识

part 1 code injection attack

phase 1

此题会调用一个test()函数

然后题目想要我们在调用getbuf()后不返回到test()中，而是进入一个touch1()的函数.

先反汇编，康康汇编码

在4017a8可以得知getbuf的缓冲区大小只有40，而在00000000004017c0为目的返回地址。

所以需要上传的序列为40个字节+4017c0 同时还需要注意使用小段法

对了hex2raw可以帮助生成字符串，然后作为参数传入ctarget

这一部分要求我们注入一小段代码进入touch2

这里我们还需要传入一个参数（值是cookie=0x59b997fa）

我们传入的序列需要实现修改%rdi的值为cookie，然后通过缓冲区溢出

这是一段需要插入的代码

但是我们应该怎样让我们的插入的代码生效呢？

You will want to position a byte representation of the address of your injected code in such a way that
ret instruction at the end of the code for getbuf will transfer control to it.

phase 3

仍然要求我们插入一段代码，将cooki作为字符串作为参数传入

When functions hexmatch and strncmp are called, they push data onto the stack, overwriting
portions of memory that held the buffer used by getbuf. As a result, you will need to be careful
where you place the string representation of your cookie.

注意到官方文档这句话，告诉我们不能把字符串存在栈中（会被覆盖），所以我们应该写入一个安全的地方通过断点，知道getbuf 申请栈空间后的 %rsp的栈地址为0x5561dca0

那么getbuf结束后应该返回到0x5561dca0 + 0x28 = 0x5561dc78

而字符串应该继续在0X5561dc78 + 48 = 0x5561dca8

cookie:0x59b997fa 对应的ascii：35 39 62 39 39 37 66 61 00 【末尾的0为截止字符】

看汇编码可以知道touch3的位置为0x4018fa
