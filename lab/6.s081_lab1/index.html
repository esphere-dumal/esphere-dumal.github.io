最近摸了很久的鱼，在打天外世界、黑魂、英灵殿、以撒还有即将出的涅米西斯。虽然摸鱼的同时也在学，但是确实没什么心情写博客。不过这段时间看的不涉及到一些核心的知识，主要是摸了摸flask的web框架，以及试着了解看了下react发现自己对前端的东西确实不是很熟。整体上是对计网应用层方面的实践，算是给计网第一步学习画了个句号。除此之外，还看了点go和c艹的东西。

但是这些玩意大多比较零散或者“上层”，感觉是时候花点时间来继续学一些系统性的东西了。

寒假的时候想写ucore的，发现自己写不动，不过现在还是得开始学os了，就打算跟着6.s081走一趟，计划4~6月主要用来看6.s081，然后暑假回家边考驾照边撸ucore.

逼逼赖赖了一些废话，可以开始lab内容了.

简介

6.s081基于xv6实现了一个类似unix的操作系统，然后实验会在已有的源码基础上进行迭代。第一个lab主要介绍了操作系统的一些基本概念，同时用代码去接触这些概念。

  • proccess and memory
  • IO and File description
  • pipes
  • file system

sleep

这个lab要求通过系统调用实现“卡住一段时间”的功能

系统提供了sleep(time)的接口，直接用就好了

pingpong

pingpong要求实现想两个进程的通信，A向B发送ping然后B回复pong

这里使用了pipe管道，通过pipe()可以得到两个文件描述符，管道是单工的，一个文件描述符用于读，另一个用于写。

这个题不难，可以初步接触多进程（fork）和进程间通信（早先在csapp里面了解了信号机制，这里又新学了管道），然后在下一道primes里面就要进一步使用了。

primes

这个题要求实现一个流水线的多进程筛数程序，是lab1里面很有分量的一个题。

  • 这个东西的实现虽然是多进程，但是通过流水线能保证同步，刚开始做的时候还在想要不要用信号同步
  • 注意资源（文件描述符）用尽的情况，pipe被fork后会占用四个描述符（父进程中两个，子进程中两个），但是事实上只需要单工通信，所以四个文件描述符中应该关掉两个。
  • 我是傻逼，一开始在用迭代的方式写递归的东西，这里开新进程最好用递归的方式。

find

实现在某个目录下查找是否存在某个文件的功能

需要看一下系统的文件实现，可以参考的源码/user/ls.c和/kernel/fs.h

特别是fs.h，这个头文件里描述了系统的文件系统，做这个lab主要看最后的dirent的实现，类似于linux下的dirent，但是少了很多东西。

然后是ls.c，主要看应该如何调用文件系统。

open – fstat – 分类讨论（直接判断文件+递归处理dirent）

感觉源码码风有点一言难尽

然后依葫芦画瓢，可以写出我们的find程序

这个题的实现很大程度是照搬的ls的源码，主要难在阅读理解

xargs

这个题要求实现一个“参数重定向”，不是很确定难点是啥
