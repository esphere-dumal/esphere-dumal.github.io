6.s081_lab2

syscall

这个lab里面包括了系统调用的实验

系统调用是用户处于用户态的时候调用需要更高权限的函数

在用户态的user/uesr.h 中定义了用户态的系统调用函数

而在内核的kernel/syscall.h中才是实际的系统操作

用户态和内核的状态由CPU中一个寄存器（是吗？）记录，当然，只有在内核态中的时候才可以修改这个状态信息。

trace

这个实验要求实现一个便于调试的工具，在程序进行系统调用的时候，输出相关信息

所以需要看看程序的实现方式

修改编译环境

在makefile中UPROGS加上$U/_trace

然后在三个文件中加上trace的定义

  • user/user.h 中 system calls 加上 int trace(int mask); 也许应该是void?
  • user/usys.pl 中加上 entry("trace") 这个脚本文件好像和makefile有一定关系，目前我还不知道
  • kernel/syscall.h 中， #define SYS_trace 22

修改原有代码

先看一下kernel/proc.h 中 proc 的实现，然后康康kernel/proc.c 中的myproc()

再是kernel/syscall.h|c ，有内核态的系统调用内容

  • kernel/sysproc.c 中实现sys_trace()，by remembering its argument in a new variable in the proc structure (see kernel/proc.h)

  • kernel/proc.c 中，修饰fork()函数，将mask状态从父进程传递到新建的子进程中

  • kernel/syscall中， 修饰syscall()函数，输出trace

    • trace格式为：411: syscall fork -> 415|| PID: syscall <func> -> ret_value

sysinfo

增加一个系统调用函数，syscall()，获取运行中的系统的基本信息

获取一个指向struct sysinfo的指针（可以再kernel/sysinfo.h中找到）

修改编译环境

  • 首先在makefile中添加$U/_sysinfotest
  • 在user/user.h 中添加sysinfo()系统调用函数
  • 在kernel/syscall.h|c 中添加 sysinfo

修改原有代码

需要将一个struct sysinfo复制返回用户态，从kernel/sysfile.c中的sys_fstat和kernel/file.c中filestat()康康怎么做(使用copyout())
