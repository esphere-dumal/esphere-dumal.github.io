Docker 网络

Docker0

容器内部地址

本机可以ping通容器内部

三个网络

docker如何处理容器网络访问

docker外部可以直接ping通docker容器

容器之间也可以彼此通信

原理

docker外部与容器内部处于同一网段

启动一个docker容器时，docker会给docker*容器分配一个ip，安装docker后就会有一个docker0网卡进行桥接模式，使用的技术是evth-pair技术

容器带来的网卡都是一对一对的

evth-pari是一对虚拟的设备接口，成对出现，一段连着协议，一段彼此相连。

docker0就像是一个路由器一样，如果容器不指定网络的情况下，都是docker0路由。

docker0通过直连NAT与物理网卡连接

小结

docker使用的Linux的桥接技术

docker网络接口都是虚拟的

–link

如果两个容器直接使用名字是不能连通的

如果有link关系，那么03可以用服务名联通02

但是02不能如此联通03

原理

在host中添加了一个tomcat02的映射

但是目前已经不建议使用–link了
