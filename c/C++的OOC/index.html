C++的OOC

Classes

object based(基于对象)

在class内部的定义完成，便自动成为inline函数。

无指针成员：complex

有指针成员：string

object oriented（面向对象）

对象间关系

继承

复合

委托b

C++演化

c++语言

c++标准库

头文件

template模板功能

常量成员函数

在函数名字的后面加const

如果函数不会改变数据内容，就可以加上const

参数传递 value|reference

传值效率低下：赋值粘贴（

指针直接传址：传址会被修改

引用底层是指针：传址+禁止修改（const）

返回值传递 value|reference

尽量传reference。如果是局部变量，那么不可以用reference 的方式返回。

传送者无需知道接受者时以reference形式接受

static

静态的数据、函数，没有thispoint

带有static的只会有一份，就是说多个类实例会共同使用一个唯一的东西

而非static的东西会有thispoint

namespace

将部分内容包装在一个命名空间里面

构造函数

函数名称与类名称相同

默认实参默认的构造值

构造函数无需返回类型和返回值

使用初值列实现的是直接初始化，而在大括号里面赋值是默认初始化再赋值，效率更低。

构造函数可以有多个—-重载overloading：函数编译后会有特殊的机器码名称，但是源码上是一样的。

拷贝构造

如果不写，编译器会默认带一份，严格地进行一项项复制。

但是这是一种浅拷贝，会存在内存泄露的问题。争对有指针的class，应该手动写深拷贝的拷贝构造函数

Big Three

普通构造
拷贝构造
拷贝赋值
析构

检测自我赋值

提高效率同时降低错误率

析构函数

不带指针的类一般不需要析构函数

当一个类死亡时，会自动调用析构函数；

友元

让一个东西可以直接拿另一个东西的private的东西

相同class的各个object互为友元

可重载操作符

链式赋值、计算

操作符是否需要一个返回值？

成员函数

为了应付用户多种形式，所以我们可能需要重载同一个操作符

非成员函数

output函数

new

stack

存在于作用域的一块内存空间

heap

system heap ， 由操作系统提供的一块global内存空间，程序可以动态分配从中获得若干区域

记住 new—-delete

array new

如果缺少配套的delete，会出现内存泄露的问题

如果对数组进行了非数组的delete，那么析构函数只会进行一次。

底层原理

new的底层使用了malloc开空间，static_cast进行转型，最后调用构造函数

特殊的内存占用

上下cookie：实现计算机内存对齐，方便内存回收

面向对象

对象的关系

继承：is-a

三种继承方式

构造先父类，再子类
析构先子类，再父类

base class 的析构函数必须是virtual，否则会出现undefined behavior

复合：has-a

什么东西里面有另一个东西

适配器|adapter

将一个高级功能的东西进行修饰

构造过程

comtainner—component

首先调用component的default构造函数，然后才执行自己的构造函数

析构过程

从外面开始析构

委托：composition by reference

这样在一开始只会有关系，但是什么时候会真正建立起has-a关系不一定

pimpl

pointer to implement

handle\body

父类指针可以指向子类

委托+继承

父类对象可以委托未来的子类

虚函数

virtual

非虚函数：derived class 被重新定义override

虚函数：derived class重新定义它，虽然我对它已经已经有了默认定义

纯虚函数：一定重新定义，因为我根本没有定义

转化函数 (conversion function)

可以将一个类型转化为一个特定的类

non-explicipt-one-argument ctor

可以将别的东西隐式转化为一个类

智能指针

reference

c++中又三种变量，value\pointer\reference.

reference的底层由指针实现

reference 是不可以改变的，因为reference可以理解为别名，reference虽然本质是指针，但是完全伪装成了指向的对象

reference 一般用于参数类型和返回类型的描述
