Qt 从0到0.5

简介

可以把Qt理解为一种开发生态环境，Qt提供了一系列工具、集成库帮助开发者完成带GUI软件的开发流程。从界面UI设计到底层代码逻辑，再到编译发布，都能使用Qt进行便捷的开发。目前，Qt主要支持c++和python。

在开发客户端的过程中，UI的设计开发和代码实现同样复杂重要，而Qt提供了Qt Designer ,将UI的设计以组件的形式可视化，方便、直观，同时易于调整参数并且与代码进行对接。

学习方向

深入学习Qt，需要有一定的c++（毕竟Qt面向对象）或 python 基础。

  • 了解Qt工程项目的文件组成
  • 了解 qt designer，便于进行GUI的设计开发，以及GUI组件与代码的对接
  • 熟悉 qt 的一些库，包括一下几个方面
    • qt 内置对象
    • 信号与槽 | 事件 （算是Qt的核心内容了
    • 各种轮子

qt 整体上是c++，但是和常规的 c++ 有所区别，需要认识其差异以及应用场合的不同

比如说 QString 和 string QFile和std::fstream

但是这些东西肯定很难记在脑子里面，所以还得学会合理使用文档、搜索引擎，让开发的过程变成学习的过程

GUI 设计

首先应该介绍一下GUI的一般组成模式

  • 最顶层的是title line，也就是进程名相关的东西
  • 顶层下面一层一般有一行菜单栏，比如说文件 编辑 段落 格式 视图 主题 帮助对，就是typora的菜单栏这样的设置一类的东西
  • 左右上下内环（一般在左侧）有工具栏，包括一些常用功能
  • 中央区域就是主视图了
  • 最下面一般有一行状态栏，提供一些信息

讲真画图软件是个很典型的例子

![image.png](Qt 从0到0.5.assets/qv5ks8VU6eTK37m.png)

Qt提供了很多组件，可以以可视化的方式生成XML文件来构成GUI界面。这些组件会成为Cpp源码里面一个ui对象的属性。

在ui designer中可以对组件进行命名和参数调整，命名是为了更好的与代码逻辑对接（也就是待会要说的信号与槽了）。

代码开发

完成了UI的设计，就要在代码层面实现各个组件之间的逻辑关系了。

qt是有很强的面向对象的性质，所以使用Qt的过程其实很多时候是在查文档，查文档，重载函数。

信号与槽

应用程序的最简单的模式就是：点击这个，呈现那个，而Qt提供了信号来对应点击这个，用槽来对应实现那个。则就是Qt中重要的信号与槽机制(signal and slot)。这个设定，很难不让人想起异常控制流中的内容，unix中的signal机制

所以我们需要给某个信号和某种处理函数关联起来

从这个notepad类很像普通的cpp类，但是多了一些东西，比如说private slots，这就是需要实现的槽，在接收到某些信号之后处理一些事务。信号的来源一般都是ui中的组件（”点击这个“），一般在构造函数中，我们将信号与槽绑定起来。（当然，可以在其他的地方绑定，根据具体实现即可，在构造函数中绑定将使信号具有普遍性效果）

connect函数实现了sender signal receiver slot的绑定，（disconnect可以解绑）。在ui组件中的信号一般都是某种QAction。

这里我们能看到Qt的两种绑定方式，其中第二种（宏定义方法）更适合带参数的non static信号。

事件机制

但是某些适合驱动应用程序运作的不一定是组件的信号，而是一些其他的机制，比如说滚动滚轮实现界面的缩放，按住ctrl激活一些功能。这时需要使用事件机制。事件类似于信号，但是无需完成信号与槽的绑定过程，而是一旦触发就会进入处理机制，可以在置于类中的protect函数，然后重载功能。比如说，上文提供的Notepad类中重载了一个wheelEvent，将对鼠标滑轮的某些反应做出反应。

Qt核心内容基本就简单介绍完了，其他内容本质上就是Qt造好的轮子，在开发时根据需求查询使用即可，而且Qt内置了非常方便的文档。多写写，自然就熟了吧
